Git 가이드

기본 개념

  • 로컬 저장소: 물리적으로 저장되어 있는 원본 파일.
  • 원격 저장소: GitHub와 같은 클라우드 기반의 저장소.
  • 브랜치: 작업의 독립적인 흐름을 만드는 방법.

Git 저장소 설정

  1. Git 저장소 초기화
    • 새로운 프로젝트를 시작할 때 필요.
    • git init
  2. 원격 저장소 연결
    • git remote add origin "GitHub Repository 주소"
    • git remote -v # 연결된 원격 저장소 확인
  3. .gitignore 설정 (선택 사항)
    • 삽입 모드로 전환: i
    • 저장하고 나가기: ESC → :wq!
    • vim .gitignore
  4. 현재 작업 상태 확인
    • 빨간 글씨: 준비 안됨
    • 초록 글씨: 준비 됨
    • git status
  5. 변경 사항 추가
    • 모든 변경 사항을 Git에 추가.
    • 특정 파일 추가: git add <파일명>
    • git add .
  6. 커밋
    • 변경된 내용을 Git 저장소에 기록.
    • git commit -m "변경 내용"
  7. 변경 사항 푸시
    • git push origin <브랜치 이름>

협업 관련 명령어

  • 브랜치 목록 확인: git branch
  • 새 브랜치 생성: git branch <새로운_브랜치 명>
  • 새 브랜치 생성과 동시에 전환: git checkout -b <새로운_브랜치 명>
  • 브랜치 전환: git checkout <브랜치 명>
  • 브랜치 병합: git merge <브랜치 명>
  • 브랜치 삭제:
    • 로컬 브랜치 삭제: git branch -d <브랜치명>
    • 원격 브랜치 삭제: git push origin --delete <브랜치명>

브랜치 충돌 해결 절차

  1. 마스터 브랜치 최신화 
    git checkout master
git fetch origin master    # 원격 저장소 변경사항 확인
git pull origin master    # 원격 변경사항 로컬에 적용
  2. 개인 브랜치 최신화 
    git checkout <개인_브랜치>
git merge master          # 마스터의 최신 내용을 개인 브랜치에 병합
  3. 충돌이 발생한 경우 
    git status               # 충돌 파일 확인
git diff                 # 충돌 내용 상세 확인
  4. 충돌 파일 수정 후 
    git add .                # 수정된 파일 스테이징
git commit -m "충돌 해결: 마스터와 개인 브랜치 병합"
git push origin <개인_브랜치>
  5. GitHub에서 Pull Request 생성 및 병합 승인 후
    • GitHub 저장소 페이지 접속
    • 'Pull requests' 탭 클릭
    • 'New pull request' 버튼 클릭
    • 브랜치 선택:
      • base: master (병합될 대상)
      • compare: <개인_브랜치> (병합할 브랜치)
    • 내용 작성 후 'Create pull request' 클릭
    • 코드 리뷰 후 'Merge pull request' 클릭
  6. 작업 완료 후 정리 
    git checkout master
git pull origin master   # 최종 변경사항 동기화

기타 유용한 명령어

  • 원격 저장소의 변경 사항 가져오기: git pull origin <브랜치 이름>
  • 사용자 이름 설정: git config --global user.name "사용자 이름"
  • 이메일 설정: git config --global user.email "이메일 주소"
  • 원격 저장소 주소 변경: git remote set-url origin "https://유저닉네임@GitHub Repository 주소"
  • 원격 저장소 클론:
    • 폴더 이름 지정: git clone "GitHub Repository 주소" "임의 이름"
    • git clone "GitHub Repository 주소"
  • 커밋 히스토리 확인:
    • git log
    • git log --oneline # 한 줄로 간단히 보기
  • 작업 되돌리기:
    • git reset --hard HEAD~1 # 직전 커밋으로 되돌리기
    • git reset --soft HEAD~1 # 커밋만 취소 (변경사항 유지)

주의사항

  • git clone이 동작하지 않을 때: 해당 경로에 동일한 이름의 폴더가 있는지 확인하세요.
  • 항상 git status로 현재 상태 확인하기
  • 중요한 작업 전 브랜치 위치 확인하기
  • 커밋 메시지는 명확하게 작성하기
  • 푸시하기 전에 항상 풀을 먼저 실행하기
  • 대규모 변경 전 반드시 백업 브랜치 생성하기
