JSP에서 바로 사용할 수 있는 도구들 설명서

내장객체가 뭔가요?

JSP 페이지에서 따로 준비할 필요 없이 바로 사용할 수 있게 미리 만들어둔 도구들입니다.

  • request (요청 정보를 담는 상자)
    • 설명: 사용자가 웹사이트에 보낸 모든 정보를 담고 있는 상자예요
    • 언제 쓰나요?: 사용자가 입력한 정보를 받거나, 브라우저 정보를 확인할 때 사용해요
      • 회원가입 시 입력한 이름을 가져올 때: String name = request.getParameter("username");
      • 사용자가 입력한 폼 데이터를 처리할 때: request.getParameter("paramName");
      • HTTP 헤더 정보를 확인할 때: request.getHeader("headerName");
      • 쿠키 정보를 가져올 때: request.getCookies();
      • 세션을 생성하거나 가져올 때: request.getSession();
  • response (응답을 보내는 도구)
    • 설명: 서버가 사용자에게 답장을 보낼 때 사용하는 도구예요
    • 언제 쓰나요?: 다른 페이지로 이동시키거나, 사용자 브라우저에 정보를 저장할 때 써요
      • 로그인 성공 후 메인 페이지로 이동: response.sendRedirect("welcome.jsp");
      • 콘텐츠 타입을 설정할 때: response.setContentType("text/html");
      • HTTP 헤더를 설정할 때: response.setHeader("headerName", "headerValue");
      • 쿠키를 추가할 때: response.addCookie(new Cookie("name", "value"));
      • 클라이언트에게 데이터를 보낼 때: response.getWriter().println("Hello World");
  • session (임시 저장소)
    • 설명: 사용자별로 필요한 정보를 잠시 보관하는 저장소예요
    • 언제 쓰나요?: 로그인 상태 유지하거나, 장바구니 기능 만들 때 사용해요
      • 로그인한 사용자 이름 저장: session.setAttribute("user", "Kim");
      • 사용자별 데이터를 저장할 때: session.setAttribute("attributeName", "value");
      • 세션을 무효화할 때: session.invalidate();
      • 세션 ID를 확인할 때: session.getId();
      • 세션 생성 시간을 확인할 때: session.getCreationTime();
  • application (웹사이트 전체 저장소)
    • 설명: 모든 사용자가 공유할 수 있는 정보를 저장하는 큰 창고예요
    • 언제 쓰나요?: 웹사이트 설정이나 모든 사용자가 봐야 하는 공지사항 같은 것을 저장할 때 써요
      • 웹사이트의 실제 경로 찾기: String path = application.getRealPath("/");
      • 애플리케이션 전역 변수를 설정할 때: application.setAttribute("globalAttribute", "value");
      • 초기화 매개변수를 가져올 때: application.getInitParameter("paramName");
      • 서버 정보를 확인할 때: application.getServerInfo();
      • 애플리케이션의 경로를 확인할 때: application.getContextPath();
  • out (화면 출력 도구)
    • 설명: 웹 페이지에 글자나 내용을 출력할 때 사용하는 도구예요
    • 언제 쓰나요?: 화면에 동적으로 내용을 보여줘야 할 때 사용해요
      • 화면에 인사말 출력: out.println("안녕하세요");
      • 출력 버퍼를 비울 때: out.flush();
      • 출력 스트림을 닫을 때: out.close();
      • 새로운 줄을 추가할 때: out.newLine();
      • 문자열을 포맷할 때: out.format("Number: %d", 10);
  • pageContext (현재 페이지 관리자)
    • 설명: 현재 보고 있는 페이지의 모든 정보를 관리하는 도구예요
    • 언제 쓰나요?: 현재 페이지에서만 필요한 정보를 저장하거나 다른 도구들을 쉽게 가져올 때 써요
      • 페이지에 메시지 저장: pageContext.setAttribute("msg", "안녕하세요");
      • 페이지 범위의 속성을 설정할 때: pageContext.setAttribute("attributeName", "value");
      • 다른 JSP 페이지를 포함할 때: pageContext.include("header.jsp");
      • 요청 객체를 가져올 때: pageContext.getRequest();
      • 응답 객체를 가져올 때: pageContext.getResponse();
  • config (초기 설정 도구)
    • 설명: 웹 페이지가 처음 시작될 때 필요한 설정 정보를 가지고 있어요
    • 언제 쓰나요?: 미리 설정해둔 값들을 읽어올 때 사용해요
      • 데이터베이스 이름 가져오기: String value = config.getInitParameter("dbName");
      • 서블릿 이름을 확인할 때: config.getServletName();
      • 서블릿 컨텍스트를 가져올 때: config.getServletContext();
      • 초기화 매개변수 이름을 가져올 때: config.getInitParameterNames();
      • 특정 초기화 매개변수를 가져올 때: config.getInitParameter("paramName");
  • page (현재 페이지 그 자체)
    • 설명: 지금 보고 있는 페이지를 나타내는 도구예요
    • 언제 쓰나요?: 거의 사용하지 않아요 (자기 자신을 가리키는 this와 같아요)
      • 페이지 정보 출력: System.out.println(page.toString());
      • 페이지 객체의 클래스 정보를 확인할 때: page.getClass();
      • 페이지 객체의 해시 코드를 확인할 때: page.hashCode();
      • 페이지 객체의 문자열 표현을 확인할 때: page.toString();
  • exception (오류 정보 담는 도구)
    • 설명: 페이지에서 문제가 발생했을 때 그 정보를 담아두는 도구예요
    • 언제 쓰나요?: 오류가 났을 때 무슨 문제가 있었는지 확인하고 싶을 때 써요 (에러 페이지에서만 사용 가능)
      • 오류 메시지 확인하기: exception.getMessage();
      • 스택 트레이스를 출력할 때: exception.printStackTrace();
      • 오류의 원인을 확인할 때: exception.getCause();
      • 오류의 지역화된 메시지를 확인할 때: exception.getLocalizedMessage();
      • 오류 객체의 문자열 표현을 확인할 때: exception.toString();
