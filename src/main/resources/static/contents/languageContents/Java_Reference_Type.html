참조 타입 (Reference Types)

제목 정의

  • 참조 타입은 객체를 저장하는 데이터 타입입니다.
  • 참조 변수는 객체의 메모리 주소를 저장하는 변수입니다.

세부 정보 정의

  1. 기본 타입: 정수, 실수, 문자 등의 값을 직접 저장합니다.
  2. 참조 타입: 객체를 가리키는 주소를 저장합니다. 예: 배열, 클래스, 인터페이스 등
  3. 메모리 사용 영역: 기본 타입은 스택에 저장되고, 참조 타입은 힙에 저장됩니다.
  4. 참조 변수의 ==, != 연산: 두 참조 변수가 같은 객체를 가리키는지 비교합니다.
  5. null: 참조 변수가 아무 객체도 가리키지 않을 때 사용됩니다. NullPointerException은 null을 참조하려 할 때 발생합니다.
  6. String 타입: 문자열을 저장하는 참조 타입입니다. String 객체는 불변입니다.

특징

  1. 참조 타입은 객체지향 프로그래밍에서 중요한 역할을 합니다.
  2. 참조 변수는 객체를 효율적으로 관리할 수 있도록 돕습니다.
  3. String 타입은 특별한 처리로 메모리에서 효율적으로 관리됩니다.

사용 이유

  1. 복잡한 데이터를 구조화하여 관리할 수 있습니다.
  2. 객체의 속성과 메서드를 활용할 수 있습니다.
  3. 메모리 관리와 성능 최적화를 위해 필요합니다.

예제 구문

public class ReferenceTypeExample {
	// 기본 타입
	int number = 10; // 기본 타입 변수

	// 참조 타입
	String text = "Hello, World!"; // String 타입 변수
	String[] array = new String[5]; // 참조 타입 배열

	// 참조 변수 비교
	String str1 = new String("Hello");
	String str2 = new String("Hello");
	System.out.println(str1 == str2); // false, 서로 다른 객체
	System.out.println(str1.equals(str2)); // true, 내용이 같음

	// null 사용
	String str3 = null; // str3는 어떤 객체도 가리키지 않음
	if (str3 == null) {
	    System.out.println("str3는 null입니다."); // 출력
	}
}

배열 (Arrays)

제목 정의

  • 배열은 동일한 타입의 여러 데이터를 저장하는 데이터 구조입니다.
  • 배열은 고정된 크기를 가지며, 인덱스를 통해 요소에 접근합니다.

세부 정보 정의

  1. 배열이란?: 여러 개의 값을 하나의 변수에 저장할 수 있는 자료형입니다.
  2. 배열 선언: 배열을 사용하기 전에 타입과 이름을 지정하여 선언합니다.
  3. 배열 생성: 메모리에 배열을 할당하여 실제 사용 준비를 합니다.
  4. 배열 길이: 배열의 요소 개수를 확인할 수 있습니다. `array.length`로 접근합니다.
  5. 명령 라인 입력: 프로그램 실행 시 배열에 값을 입력받을 수 있습니다.
  6. 다차원 배열: 배열 안에 배열을 포함한 형태로, 여러 차원의 데이터를 저장합니다.
  7. 객체를 참조하는 배열: 객체 타입의 배열로, 여러 객체를 저장할 수 있습니다.
  8. 배열 복사: 배열의 내용을 다른 배열로 복사할 수 있습니다.
  9. 향상된 for문: 배열의 요소를 쉽게 반복하여 접근할 수 있는 구문입니다.

특징

  1. 배열은 고정된 크기를 가지므로, 크기를 변경할 수 없습니다.
  2. 모든 요소는 동일한 데이터 타입이어야 합니다.
  3. 인덱스는 0부터 시작하며, 범위를 초과할 경우 오류가 발생합니다.

사용 이유

  1. 같은 타입의 데이터를 효율적으로 관리하기 위해 필요합니다.
  2. 데이터에 대한 반복적인 작업을 쉽게 수행할 수 있습니다.
  3. 메모리를 효율적으로 사용할 수 있습니다.

예제 구문

// 배열 선언
int[] numbers; // 정수형 배열 선언

// 배열 생성
numbers = new int[5]; // 크기 5의 배열 생성

// 배열 초기화
numbers[0] = 10;
numbers[1] = 20;

// 배열 길이
System.out.println("배열 길이: " + numbers.length); // 5 출력

// 다차원 배열
int[][] matrix = new int[3][3]; // 3x3 배열 생성

// 객체를 참조하는 배열
String[] names = {"Alice", "Bob", "Charlie"}; // 문자열 배열

// 배열 복사
int[] copy = numbers.clone(); // 배열 복사

// 향상된 for문
for (int num : numbers) {
    System.out.println(num); // 배열 요소 출력
}

열거 타입(Enumeration Type)

제목 정의

  • 열거타입은 미리 정의된 상수 집합을 나타내는 자료형입니다.
  • 주로 관련된 상수들을 그룹화하여 사용합니다.

세부 정보 정의

  1. 열거 타입 선언: 열거타입을 정의하여 특정한 이름의 집합을 만듭니다.
  2. 열거 타입 변수: 선언된 열거타입의 값을 저장할 수 있는 변수입니다.

특징

  1. 열거타입은 코드의 가독성을 높이고 오류를 줄여줍니다.
  2. 각 상수는 고유한 이름을 가지며, 이를 통해 값의 의미를 명확하게 전달합니다.
  3. 열거타입은 타입 안전성을 제공합니다. 즉, 잘못된 값이 할당되지 않도록 합니다.

사용 이유

  1. 특정 값의 집합을 정의하고 관리하기 위해 필요합니다.
  2. 코드에서 사용되는 상수를 중앙 집중식으로 관리할 수 있습니다.
  3. 가독성을 높이고 유지보수를 용이하게 합니다.

예제 구문

// 열거타입 선언
enum Day {
    SUNDAY, MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY
}

// 열거 타입 변수
Day today = Day.MONDAY; // 오늘의 요일 설정

// 열거타입 사용 예
if (today == Day.MONDAY) {
    System.out.println("오늘은 월요일입니다.");
}
