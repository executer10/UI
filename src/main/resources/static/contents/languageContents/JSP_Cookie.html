1. 쿠키의 기본 개념

1.1 쿠키란 무엇인가?

쿠키는 웹 서버가 클라이언트(브라우저)에 저장하는 작은 데이터 파일입니다. 주요 목적은 상태 정보 유지와 사용자 경험 개선입니다.

1.2 HTTP 상태 유지의 필요성

  • HTTP 프로토콜은 본질적으로 비상태(Stateless) 프로토콜
  • 페이지 이동 시 이전 상태 정보 손실 문제 해결
  • 사용자 맞춤형 서비스 제공을 위한 필수 기술

2. 쿠키의 주요 특징

2.1 장점

  • 간편한 상태 정보 저장
  • 클라이언트 측 데이터 유지 가능
  • 개인화된 웹 서비스 구현

2.2 제한사항

  • 쿠키당 최대 4KB 저장 가능
  • 도메인당 최대 20개 쿠키 제한
  • 보안에 취약할 수 있음

3. 쿠키 구현 예시

3.1 쿠키 생성 예제

// 사용자 로그인 쿠키 생성
Cookie 로그인쿠키 = new Cookie("회원아이디", "홍길동");
로그인쿠키.setMaxAge(24 * 60 * 60); // 1일 유효 
로그인쿠키.setHttpOnly(true);      // XSS 방지
response.addCookie(로그인쿠키);

// 실행 결과: 
// - "회원아이디" 쿠키 생성
// - 1일 동안 유효
// - JavaScript 접근 차단

3.2 쿠키 읽기 예제

// 쿠키 배열 가져오기
Cookie[] 쿠키목록 = request.getCookies();
for (Cookie 단일쿠키 : 쿠키목록) {
    if (단일쿠키.getName().equals("회원아이디")) {
        String 회원정보 = 단일쿠키.getValue();
        // 회원정보 처리
    }
}

// 실행 결과:
// - 모든 쿠키 순회
// - "회원아이디" 쿠키 값 추출
