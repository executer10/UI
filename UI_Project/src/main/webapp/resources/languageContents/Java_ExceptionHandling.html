예외 클래스

설명

  • 예외(Exception)는 프로그램 실행 중 발생하는 비정상적인 상황
  • 예외 클래스는 이러한 예외 상황을 나타내는 객체의 클래스

특징

  1. 모든 예외 클래스는 java.lang.Exception 클래스를 상속
  2. 실행 예외(Runtime Exception)는 RuntimeException 클래스를 상속
  3. 주요 실행 예외 클래스:
    • NullPointerException: 객체 참조가 없는 상태에서 메소드 호출
    • ArrayIndexOutOfBoundsException: 배열의 범위를 벗어난 인덱스 접근
    • NumberFormatException: 숫자로 변환할 수 없는 문자열을 숫자로 변환 시도
    • ClassCastException: 부적절한 클래스 형변환

사용 이유

  1. 프로그램 안정성 향상: 예외 처리를 통해 비정상 종료 방지
  2. 디버깅 용이성: 예외 정보를 통해 문제 원인 파악 쉬움
  3. 코드 가독성: 정상 실행 코드와 예외 처리 코드 분리

예제 구문

public class ExceptionExample {
    public static void main(String[] args) {
        // NullPointerException 예제
        try {
            String str = null;
            System.out.println(str.length());
        } catch (NullPointerException e) {
            System.out.println("NullPointerException 발생: " + e.getMessage());
        }

        // ArrayIndexOutOfBoundsException 예제
        try {
            int[] arr = new int[3];
            System.out.println(arr[3]);
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("ArrayIndexOutOfBoundsException 발생: " + e.getMessage());
        }

        // NumberFormatException 예제
        try {
            int num = Integer.parseInt("abc");
        } catch (NumberFormatException e) {
            System.out.println("NumberFormatException 발생: " + e.getMessage());
        }

        // ClassCastException 예제
        try {
            Object obj = new Integer(123);
            String str = (String) obj;
        } catch (ClassCastException e) {
            System.out.println("ClassCastException 발생: " + e.getMessage());
        }
    }
}
