조건문

제목 정의

  • 조건문은 특정 조건에 따라 코드의 실행 흐름을 제어하는 구문입니다.
  • 주로 if문과 switch문이 사용됩니다.

세부 정보 정의

  1. if문: 주어진 조건이 참일 때 특정 코드를 실행합니다.
  2. if-else문: 조건이 참일 때와 거짓일 때 각각 다른 코드를 실행합니다.
  3. if-else if-else문: 여러 조건을 검사하여 그 중 하나에 해당하는 코드를 실행합니다.
  4. switch문: 여러 경우 중 하나에 해당하는 코드를 실행합니다. 보통 특정 변수의 값에 따라 분기합니다.

특징

  1. 조건문은 프로그램의 흐름을 동적으로 변경할 수 있습니다.
  2. 조건은 불리언 값(참 또는 거짓)으로 평가됩니다.
  3. 각 조건문은 다양한 형태로 중첩하여 사용할 수 있습니다.

사용 이유

  1. 프로그램의 로직을 구현하기 위해 필요합니다.
  2. 입력 값이나 상태에 따라 서로 다른 처리를 할 수 있습니다.
  3. 효율적으로 여러 경우를 처리할 수 있습니다.

예제 구문 및 코드

// if문
if (age >= 18) {
    System.out.println("성인입니다.");
}

// if-else문
if (score >= 60) {
    System.out.println("합격입니다.");
} else {
    System.out.println("불합격입니다.");
}

// if-else if-else문
if (grade == 'A') {
    System.out.println("우수입니다.");
} else if (grade == 'B') {
    System.out.println("보통입니다.");
} else {
    System.out.println("저조입니다.");
}

// switch문
switch (day) {
    case 1:
        System.out.println("월요일");
        break;
    case 2:
        System.out.println("화요일");
        break;
    default:
        System.out.println("주말입니다.");
}

반복문

제목 정의

  • 반복문은 특정 조건이 참인 동안 코드를 반복 실행하는 구문입니다.
  • 주로 for문, while문, do-while문이 사용됩니다.

세부 정보 정의

  1. for문: 정해진 횟수만큼 반복 실행합니다. 반복 변수와 조건을 설정합니다.
  2. while문: 주어진 조건이 참인 동안 반복 실행합니다. 조건이 먼저 평가됩니다.
  3. do-while문: 코드를 먼저 실행한 후 조건을 평가합니다. 최소 한 번은 실행됩니다.

특징

  1. 반복문은 코드의 반복 실행을 간소화합니다.
  2. 조건에 따라 반복을 조절할 수 있습니다.
  3. 무한 반복이 발생할 수 있으므로 주의해야 합니다.

사용 이유

  1. 같은 작업을 여러 번 수행해야 할 때 필요합니다.
  2. 데이터의 집합을 처리할 때 유용합니다.
  3. 코드의 중복을 줄이고 가독성을 높입니다.

예제 구문 및 코드

                    // for문
                    for (int i = 0; i < 5; i++) {
                        System.out.println(i); // 0부터 4까지 출력
                    }
                    
                    // while문
                    int count = 0;
                    while (count < 5) {
                        System.out.println(count); // 0부터 4까지 출력
                        count++;
                    }
                    
                    // do-while문
                    int num = 0;
                    do {
                        System.out.println(num); // 0 출력
                        num++;
                    } while (num < 1); // 조건이 거짓이더라도 한 번 실행

향상된 for문 또는 for-each문

설명

  • 향상된 for문 (또는 for-each문)은 배열 또는 컬렉션의 모든 요소를 순차적으로 반복하기 위해 사용되는 간편한 반복문입니다.
  • 배열이나 컬렉션의 크기를 신경 쓰지 않고도, 각 요소에 접근할 수 있습니다.
  • 일반적인 for문보다 코드가 간결하며, 요소에 대한 읽기 작업에 주로 사용됩니다.

주요 특징

  1. 간결함: 배열이나 컬렉션의 모든 요소를 한 번에 처리할 수 있어 코드가 간단해집니다.
  2. 읽기 전용: 요소를 수정할 필요가 없고, 반복할 때 주로 읽기 작업을 수행합니다.
  3. 자동화된 순회: 인덱스를 관리할 필요 없이 자동으로 배열의 각 요소에 접근합니다.

사용 이유

  1. 배열이나 리스트의 모든 요소를 한 번에 처리할 때
  2. 인덱스를 직접 관리하지 않고, 간편하게 요소를 순회할 때
  3. 읽기 작업이 주로 필요할 때

예제 구문 및 코드

// 향상된 for문 (for-each문) 예제
public class Main {
    public static void main(String[] args) {
        int[] numbers = {1, 2, 3, 4, 5};

        // 배열의 각 요소를 출력하는 향상된 for문
        for (int number : numbers) {
            System.out.println("Number: " + number);
        }
    }
}

레이블이 지정된 break 문

설명

  • 레이블이 지정된 break문은 중첩된 반복문에서 특정 레이블이 지정된 바깥쪽 반복문을 종료할 때 사용됩니다.
  • 일반적인 break문은 가장 가까운 반복문만 종료하지만, 레이블을 사용하면 더 바깥쪽 반복문을 제어할 수 있습니다.
  • 레이블은 반복문 앞에 선언하며, break문은 해당 레이블을 참조하여 특정 반복문을 종료합니다.

특징

  1. 중첩된 반복문 제어: 중첩된 여러 반복문을 레이블을 사용하여 원하는 단계에서 종료할 수 있습니다.
  2. 가독성 향상: 코드가 복잡해지는 중첩된 반복문에서 레이블을 사용해 명확하게 제어할 수 있습니다.
  3. 유연한 흐름 제어: 일반적인 흐름 제어보다 더 유연하게 특정 반복문을 빠져나올 수 있습니다.

사용 이유

  1. 중첩된 반복문에서 특정 조건에 따라 여러 반복문을 한 번에 종료할 때
  2. 단순한 break로는 해결하기 어려운 흐름을 제어할 때
  3. 복잡한 중첩된 구조에서 더 가독성 있는 코드를 작성하고 싶을 때

예제 구문 및 코드

// 레이블이 지정된 break문 예제
public class Main {
    public static void main(String[] args) {
        // 레이블 지정
        outerLoop:
        for (int i = 1; i <= 3; i++) {
            for (int j = 1; j <= 3; j++) {
                System.out.println("i: " + i + ", j: " + j);
                // 특정 조건에서 바깥쪽 반복문 종료
                if (i == 2 && j == 2) {
                    break outerLoop;
                }
            }
        }
    }
}
