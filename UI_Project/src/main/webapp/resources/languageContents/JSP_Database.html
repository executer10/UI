목차

로그인 시스템 구조도
  • 데이터 계층
    • 회원정보 관리 (MemberDTO)
    • 데이터 접근 객체 (MemberDAO)
  • 데이터베이스 연결
    • 기본 JDBC 연결 (JDBConnect)
    • 커넥션 풀 관리 (DBConnPool)
  • 사용자 인터페이스
    • 로그인 화면 구현
    • 로그인 처리 로직
    • 로그아웃 기능
    • 메뉴 네비게이션

세부 내용

  1. 데이터 계층 구현
    • 회원정보 DTO (Data Transfer Object)
      • 필수 회원 정보: 아이디, 비밀번호, 이름, 가입일자
      • 데이터 보안: private 접근제어자 사용
      • 데이터 접근: getter/setter 메소드 구현
    • 회원정보 DAO (Data Access Object)
      • 데이터베이스 연결: JDBConnect 상속
      • 회원 인증: PreparedStatement 활용
      • 보안 강화: SQL 인젝션 방지
  2. 데이터베이스 연결 관리
    • JDBC 직접 연결
      • 연결 방식: 기본, 매개변수, ServletContext
      • 자원 관리: Connection, Statement 등
      • 예외 처리: try-catch 구문 활용
    • 커넥션 풀 활용
      • 설정: JNDI 컨텍스트 초기화
      • 효율성: 자동 커넥션 반환
      • 모니터링: 연결 상태 로깅
  3. 사용자 인터페이스 구현
    • 로그인 화면 (로그인폼.jsp)
      • 입력 검증: JavaScript 유효성 검사
      • 화면 구성: 조건부 렌더링
      • 오류 처리: 메시지 표시 기능
    • 로그인 처리 (로그인처리.jsp)
      • 데이터 처리: 폼 데이터 수신
      • 인증 절차: DAO 활용
      • 세션 관리: 사용자 정보 저장

특징

  1. 보안 강화 기능
    • SQL 인젝션 방지: PreparedStatement 사용
    • 세션 기반 인증: 로그인 상태 관리
    • 입력값 검증: 클라이언트/서버 이중 검사
  2. 효율적인 리소스 관리
    • 커넥션 풀 활용: 데이터베이스 연결 최적화
    • 자원 해제: 체계적인 close() 처리
    • 메모리 관리: 누수 방지 구조
  3. 사용자 경험 최적화
    • 직관적 인터페이스: 한글 메시지 활용
    • 오류 처리: 명확한 피드백 제공
    • 동적 메뉴: 로그인 상태 반영
