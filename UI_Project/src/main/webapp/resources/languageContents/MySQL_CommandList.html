1. DDL (Data Definition Language) - 데이터 정의어
[데이터베이스 객체 구조를 정의]
  • CREATE: 새로운 데이터베이스 객체 생성 (테이블, 뷰, 인덱스 등)
    • CREATE DATABASE: 데이터베이스 생성
    • CREATE TABLE: 테이블 생성
    • CREATE INDEX: 인덱스 생성
[데이터 타입]
  • 숫자형
    • 정수: INT, TINYINT, SMALLINT, MEDIUMINT, BIGINT
    • 실수: FLOAT, DOUBLE, DECIMAL(정확한 소수)
  • 문자형
    • 고정길이: CHAR
    • 가변길이: VARCHAR, TEXT
    • 대용량: TINYTEXT, MEDIUMTEXT, LONGTEXT
  • 날짜/시간형
    • DATE: 날짜(YYYY-MM-DD)
    • TIME: 시간(HH:MM:SS)
    • DATETIME/TIMESTAMP: 날짜와 시간
    • YEAR: 연도
  • 이진데이터
    • BINARY, VARBINARY: 고정/가변 길이 이진 데이터
    • BLOB 유형: TINYBLOB, BLOB, MEDIUMBLOB, LONGBLOB
  • 기타: ENUM(열거형), SET(집합), JSON
[제약조건]
  • NOT NULL: NULL 값 허용하지 않음
  • UNIQUE: 중복값 허용하지 않음
  • PRIMARY KEY: 기본키 지정 (NOT NULL + UNIQUE)
  • FOREIGN KEY: 외래키 지정 (다른 테이블 참조)
  • CHECK: 입력값 범위 제한
  • DEFAULT: 기본값 설정
  • AUTO_INCREMENT: 자동 증가
[테이블 옵션]
  • ENGINE: 스토리지 엔진 설정
    • InnoDB: 트랜잭션 지원 (기본)
    • MyISAM: 빠른 읽기 작업
  • CHARACTER SET: 문자셋 설정 (예: utf8mb4)
  • COLLATE: 정렬 규칙 설정
  • COMMENT: 테이블 주석
[ALTER: 기존 객체 구조 변경]
  • ADD: 컬럼/제약조건 추가
  • MODIFY: 컬럼 정의 수정
  • CHANGE: 컬럼명과 정의 수정
  • DROP: 컬럼/제약조건 삭제
  • RENAME: 컬럼명 변경
  • ALTER COLUMN: 컬럼 속성 변경
[DROP: 객체 삭제]
  • CASCADE: 연관된 객체도 함께 삭제
  • RESTRICT: 참조하는 객체가 있으면 삭제 거부
[뷰 관련]
  • CREATE VIEW: 뷰 생성
  • ALTER VIEW: 뷰 수정
  • DROP VIEW: 뷰 삭제
[파티션 관련]
  • PARTITION BY: 파티션 정의
    • RANGE: 범위 기반 파티셔닝
    • LIST: 목록 기반 파티셔닝
    • HASH: 해시 기반 파티셔닝
    • KEY: 키 기반 파티셔닝
  • ADD PARTITION: 파티션 추가
  • DROP PARTITION: 파티션 삭제
2. DML (Data Manipulation Language) - 데이터 조작어
[기본 명령어]
  • SELECT: 데이터 조회
    • DISTINCT: 중복 제거
    • AS: 별칭 지정
    • UNION/UNION ALL: 결과 집합 결합
    • CASE WHEN: 조건부 표현식
  • INSERT: 데이터 삽입
    • INSERT INTO: 테이블에 데이터 삽입
    • INSERT IGNORE: 중복 무시
    • ON DUPLICATE KEY UPDATE: 중복 시 수정
  • UPDATE: 데이터 수정
    • SET: 수정할 값 지정
  • DELETE: 데이터 삭제
    • DELETE FROM: 테이블에서 데이터 삭제
  • MERGE: 조건부 삽입/수정
  • REPLACE: 레코드 업데이트 또는 삽입
  • CALL: 저장 프로시저/Java 서브프로그램 호출
  • EXPLAIN: 쿼리 실행 계획 표시
[집계 함수]
  • COUNT(): 행 개수 계산
  • SUM(): 합계 계산
  • AVG(): 평균값 계산
  • MAX(): 최대값 검색
  • MIN(): 최소값 검색
[주요 절(Clause)]
  • WHERE: 조건 필터링
    • 비교 연산자: =, <>, >, <,>=, <=
    • IN, NOT IN: 목록 포함 여부
    • BETWEEN AND: 범위 지정
    • LIKE, NOT LIKE: 패턴 매칭
    • IS NULL, IS NOT NULL: NULL 값 확인
  • GROUP BY: 그룹화
    • WITH ROLLUP: 총계 추가
  • HAVING: 그룹 조건
  • ORDER BY: 정렬
    • ASC: 오름차순
    • DESC: 내림차순
  • LIMIT: 결과 개수 제한
    • OFFSET: 시작 위치 지정
  • JOIN: 테이블 결합
    • INNER JOIN: 내부 조인
    • LEFT JOIN: 왼쪽 외부 조인
    • RIGHT JOIN: 오른쪽 외부 조인
    • FULL JOIN: 전체 외부 조인
    • CROSS JOIN: 교차 조인
    • NATURAL JOIN: 자연 조인
3. DCL (Data Control Language) - 데이터 제어어
[권한 관리]
  • GRANT: 권한 부여
  • REVOKE: 권한 회수
  • DENY: 권한 명시적 거부 (DBMS 별로 상이)
[사용자 관리]
  • CREATE USER: 사용자 생성
  • ALTER USER: 사용자 수정
  • DROP USER: 사용자 삭제
4. TCL (Transaction Control Language) - 트랜잭션 제어어
  • BEGIN/START TRANSACTION: 트랜잭션 시작
  • COMMIT: 트랜잭션 확정
  • ROLLBACK: 트랜잭션 취소
  • SAVEPOINT: 중간저장점 생성
5. 시스템 관리
[시스템 명령어]
  • help (\?): 도움말 표시
  • clear (\c): 현재 입력 취소
  • connect (\r): 서버 재연결
  • quit (\q): MySQL 종료
  • status (\s): 서버 상태 확인
  • source: SQL 파일 실행
  • use: 데이터베이스 선택
[백업/복구]
  • mysqldump: 데이터베이스 백업
  • mysqlimport: 데이터 가져오기
  • LOAD DATA INFILE: 파일에서 데이터 로드
  • SELECT INTO OUTFILE: 데이터를 파일로 내보내기
[테이블 관리]
  • ANALYZE TABLE: 테이블 분석
  • CHECK TABLE: 테이블 검사
  • OPTIMIZE TABLE: 테이블 최적화
  • REPAIR TABLE: 테이블 복구
